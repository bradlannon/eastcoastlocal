    1. Install packages: `npm install leaflet.heat` and `npm install -D @types/leaflet.heat`

    2. Create `src/components/map/HeatmapLayer.tsx` following the exact pattern from 04-RESEARCH.md Pattern 1 (Imperative Leaflet Plugin Wrapping):
       - `'use client'` directive
       - Import `useMap` from `react-leaflet`, `L` from `leaflet`, `import 'leaflet.heat'`
       - Props: `points: HeatPoint[]`, `visible: boolean`
       - First `useEffect([map])`: create `L.heatLayer([], { radius: 35, blur: 20, maxZoom: 12, gradient: { 0.2: '#3b82f6', 0.5: '#f59e0b', 0.8: '#ef4444' } })`. Cleanup calls `map.removeLayer()` and nulls ref. Use `@ts-expect-error` for `L.heatLayer` since leaflet.heat augments L at runtime.
       - Second `useEffect([map, points, visible])`: if not visible, call `map.removeLayer()` and return. Otherwise, convert points to `[lat, lng, intensity][]`, call `heatRef.current.setLatLngs(latlngs)`, add to map if not already added, call `redraw()`.
       - Return `null` (render-nothing component).
       - Export `HeatPoint` interface for reuse.

    3. Verify SSR build gate: Run `npx next build`. This MUST pass without `window is not defined` errors. HeatmapLayer.tsx is safe because it will only be imported from MapClient.tsx which is behind the `dynamic(..., { ssr: false })` boundary in MapClientWrapper.tsx.

    If `next build` fails with `window is not defined`:
    - Move the `import 'leaflet.heat'` inside the first `useEffect` as a dynamic `import('leaflet.heat')` call
    - Re-run `npx next build` to confirm fix

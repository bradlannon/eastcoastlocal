<feature>
  <name>Timelapse Utility Functions</name>
  <files>src/lib/timelapse-utils.ts, src/lib/timelapse-utils.test.ts</files>
  <behavior>
    Constants:
    - TOTAL_DAYS = 30, BLOCKS_PER_DAY = 4, TOTAL_STEPS = 120, STEP_SIZE = 1/120, WINDOW_HOURS = 24

    positionToTimestamp(position, referenceDate):
    - position=0 -> referenceDate
    - position=1 -> referenceDate + 30 days
    - position=0.5 -> referenceDate + 15 days

    positionToBlockName(position):
    - Step 0 (position ~0) -> "Morning"
    - Step 1 (position 1/120) -> "Afternoon"
    - Step 2 (position 2/120) -> "Evening"
    - Step 3 (position 3/120) -> "Night"
    - Step 4 (position 4/120) -> "Morning" (next day cycle)

    filterByTimeWindow(events, centerMs, windowHours=24):
    - Events within +/-12h of centerMs -> included
    - Events outside window -> excluded
    - Empty events array -> returns empty array
    - windowHours=0 -> returns nothing (edge case)

    computeVenueHeatPoints(events):
    - 3 events at venue A, 1 at venue B -> venue A intensity=1.0, venue B intensity=max(0.15, 1/3)
    - Events with null lat/lng -> skipped
    - Empty events -> returns empty array
    - Single event at single venue -> intensity=max(0.15, 1) = 1.0
    - Multiple events same venue -> grouped, count normalized
  </behavior>
  <implementation>
    Implement all functions as documented in 04-RESEARCH.md Pattern 3 (Normalized Float Time Position). Follow the exact code from the research — it has been verified. Use date-fns for timestamp math. Export the HeatPoint interface and BlockName type for downstream consumers.
  </implementation>
</feature>

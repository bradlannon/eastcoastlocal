  <action>
Create server actions file at src/app/admin/discovery/actions.ts with 'use server' directive.

**approveCandidate(formData: FormData): Promise<void>**
1. Extract `id` from formData, parse as integer. If invalid, return early.
2. Call `promoteSource(id)` from `@/lib/scraper/promote-source`. Wrap in try/catch — if it throws (not found, not pending), log the error but still proceed to revalidate.
3. Call `revalidatePath('/admin/discovery')` then `redirect('/admin/discovery')`.

Note: promoteSource already handles all the promotion logic (create venue, create scrape_source, update status to 'approved', set timestamps). No need to duplicate any of that.

**rejectCandidate(prevState: { error?: string }, formData: FormData): Promise<{ error?: string }>**
1. Extract `id` from formData, parse as integer. If invalid, return `{ error: 'Invalid ID' }`.
2. Extract optional `reason` from formData, trim it. Empty string is fine (reason is optional per user decision).
3. Update discovered_sources: set `status = 'rejected'`, `reviewed_at = new Date()`. If reason is non-empty, store it — but the schema has no `rejection_reason` column, so store the reason by updating `raw_context` to append `"\n\n--- Rejection Reason ---\n{reason}"` to preserve existing raw_context. Alternatively, since the schema doesn't have a reason field and adding columns is out of scope, simply update status and reviewed_at. The rejection reason can be logged to console. Per user decision, the reason is recorded "in the database" — so append it to raw_context as the simplest approach without a migration.
4. Call `revalidatePath('/admin/discovery')` then `redirect('/admin/discovery')`.

Import: `eq` from drizzle-orm, `db` from @/lib/db/client, `discovered_sources` from @/lib/db/schema, `promoteSource` from @/lib/scraper/promote-source, `revalidatePath` from next/cache, `redirect` from next/navigation.

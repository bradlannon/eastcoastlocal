  <name>Task 1: Create orchestrator and cron route with tests</name>
  <files>

  <behavior>
    Orchestrator:
    - runScrapeJob queries all enabled scrape_sources with their venue relation
    - For venue_website sources: calls fetchAndPreprocess -> extractEvents -> for each event: check if venue has lat/lng, if not geocode -> upsertEvent
    - For eventbrite sources: calls scrapeEventbrite(source)
    - For bandsintown sources: calls scrapeBandsintown(source)
    - On success per source: updates last_scraped_at=now, last_scrape_status='success'
    - On error per source: logs error, updates last_scraped_at=now, last_scrape_status='failure', continues to next source
    - Never aborts the full run for a single source failure

    Cron route:
    - GET without Authorization header returns 401
    - GET with wrong Bearer token returns 401
    - GET with correct Bearer ${CRON_SECRET} calls runScrapeJob and returns 200 { success: true, timestamp }
    - GET returns 500 { success: false, error } if runScrapeJob throws
  </behavior>
  <action>
    1. Create `src/lib/scraper/orchestrator.ts`:
       - Export `runScrapeJob(): Promise<void>`
       - Query: `db.query.scrape_sources.findMany({ where: eq(scrape_sources.enabled, true) })` — note: the Neon HTTP driver with Drizzle may not support `with: { venue: true }` relational query without explicit relations defined in schema. Instead, query scrape_sources then query the venue separately: `const venue = await db.query.venues.findFirst({ where: eq(venues.id, source.venue_id) })`.
       - For each source, wrap in try/catch:
         - `venue_website`: call `fetchAndPreprocess(source.url)`, then `extractEvents(pageText, source.url)`, then for each event: if venue lacks lat/lng, call `geocodeAddress(venue.address + ', ' + venue.city + ', ' + venue.province + ', Canada')` and update venue record with coordinates. Then `upsertEvent(source.venue_id, event, source.url)`.
         - `eventbrite`: call `scrapeEventbrite(source)`
         - `bandsintown`: call `scrapeBandsintown(source)`
       - On success: `db.update(scrape_sources).set({ last_scraped_at: new Date(), last_scrape_status: 'success' }).where(eq(scrape_sources.id, source.id))`
       - On catch: log error, update `last_scrape_status: 'failure'`, continue loop

    2. Create `src/app/api/cron/scrape/route.ts`:
       - Export `maxDuration = 60` (Hobby plan limit; comment noting Pro allows 300)
       - Export `GET(request: Request)` handler
       - Check `request.headers.get('authorization') !== Bearer ${process.env.CRON_SECRET}` — return 401 if mismatch
       - Call `runScrapeJob()`, return `Response.json({ success: true, timestamp: new Date().toISOString() })`
       - Catch errors: return `Response.json({ success: false, error: String(err) }, { status: 500 })`

    3. Write tests FIRST (TDD):
       - `route.test.ts`: Mock `@/lib/scraper/orchestrator` module. Set `process.env.CRON_SECRET = 'test-secret'`. Test: no auth header -> 401, wrong token -> 401, correct token -> runScrapeJob called + 200 returned, runScrapeJob throws -> 500 returned.
       - Note: Orchestrator integration testing is covered by the manual checkpoint below — unit tests for the route are sufficient here.

    Mock pattern for route tests:
    ```typescript
    jest.mock('@/lib/scraper/orchestrator', () => ({
      runScrapeJob: jest.fn().mockResolvedValue(undefined),
    }));
    ```

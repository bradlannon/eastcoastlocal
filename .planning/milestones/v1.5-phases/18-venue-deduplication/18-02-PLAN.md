<task type="auto">
  <name>Task 3: Create backfill CLI script for existing venue duplicates</name>
  <files>scripts/venue-dedup-backfill.ts</files>
  <action>
    Create `scripts/venue-dedup-backfill.ts` — a one-off CLI script that scans existing venues for duplicates.

    **Usage:**
    ```bash
    npx tsx scripts/venue-dedup-backfill.ts              # dry-run (default)
    npx tsx scripts/venue-dedup-backfill.ts --execute     # actually merge
    ```

    **Implementation:**
    1. Parse CLI args: check for `--execute` flag. Default is dry-run.
    2. Load all venues from DB grouped by city (using `ilike` for city normalization is not needed here — just group by city column value).
    3. For each city group with 2+ venues:
       - Compare every pair using `scoreVenueCandidate` from `venue-dedup.ts`
       - Collect merge decisions and review decisions
    4. In dry-run mode:
       - Print a summary table to stdout: `incoming | canonical | name_score | distance_m | decision`
       - Print totals: "X merge candidates, Y review candidates, Z kept separate"
       - Exit without DB writes
    5. In execute mode:
       - For each merge decision:
         a. Reassign all events from duplicate venue to canonical: `UPDATE events SET venue_id = canonical_id WHERE venue_id = duplicate_id` (the dedup key unique index will reject events that already exist on canonical — wrap in try/catch and skip on conflict)
         b. Reassign any scrape_sources from duplicate to canonical (per Pitfall 3)
         c. Delete the duplicate venue row
         d. Log to venue_merge_log table
       - For each review decision:
         a. Insert into venue_merge_candidates table (if not already present)
       - Print summary of actions taken

    **Conflict handling on event reassignment:** When moving events from duplicate to canonical, some events may violate the `(venue_id, event_date, normalized_performer)` unique constraint because the same event already exists on canonical (from a different source). Use a per-event UPDATE wrapped in try/catch — skip silently on unique constraint violation, delete the orphaned event from the duplicate instead.

    **Import pattern:** Use relative imports since this is a script, not a Next.js page:
    ```typescript
    import { db } from '../src/lib/db/client';
    import { venues, events, scrape_sources, venueMergeLog, venueMergeCandidates } from '../src/lib/db/schema';
    import { scoreVenueCandidate } from '../src/lib/scraper/venue-dedup';
    ```
    Or use the `@/` alias if tsx respects the tsconfig paths (it does in this project).

- Phase 22 schema columns → Phase 23 stageCandidate writes them → Phase 22 promoteSource carries to venues

- Phase 25 discovery_runs table → all 6 cron routes write success + error rows
- Phase 25 batchApproveCandidate → form in DiscoveryList.tsx

- discovery_runs.queued_pending mapped correctly from all three discoverer result types
- Admin dashboard queries discovery_runs for stat card + recent 10 table

### Integration Gaps (3 items — all low severity)

1. **INT-01: no_website stubs invisible in admin UI** — Discovery page shows pending/approved/rejected tabs only. Stubs at status='no_website' never appear. promoteSource handles them correctly; the gap is UI visibility only.

2. **INT-02: phone column never populated** — Phase 22 added phone to schema. Phase 23 FIELD_MASK doesn't request phone fields. Column is dead weight / reserved for future use.

3. **INT-03: GEMINI_AUTO_APPROVE defined 3 times** — places-discoverer hardcodes 0.9; orchestrator and reddit-discoverer read env var. No runtime impact at defaults but env override doesn't affect Places threshold.

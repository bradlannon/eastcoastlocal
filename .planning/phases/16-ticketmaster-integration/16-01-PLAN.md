  <action>
    Create `src/lib/scraper/ticketmaster.ts` following the eventbrite.ts pattern exactly:

    1. **TM types:** Define `TmEvent` and `TmResponse` interfaces matching the TM Discovery API v2 response shape (see RESEARCH.md Response Field Map).

    2. **`scrapeTicketmaster(source: ScrapeSource): Promise<void>`:**
       - Parse stateCode from `source.url.replace('ticketmaster:province:', '')`
       - Read `process.env.TICKETMASTER_API_KEY`
       - Build URL params: apikey, countryCode=CA, stateCode, startDateTime (today T00:00:00Z), endDateTime (today+30d T23:59:59Z), size=200
       - Fetch `https://app.ticketmaster.com/discovery/v2/events.json?${params}`
       - Throw on non-OK response with status + stateCode in message
       - Iterate `data._embedded?.events ?? []`
       - Skip events without `_embedded.venues[0]`
       - For each event: call `findOrCreateVenue()` to get venueId, then call `upsertEvent(venueId, extractedEvent, event.url)`
       - Performer: `event._embedded?.attractions?.[0]?.name ?? event.name`
       - Date: `event.dates.start.localDate` (string "YYYY-MM-DD")
       - Time: if `timeTBA` true, null; else `event.dates.start.dateTime?.slice(11, 16) ?? null`
       - Price: `priceRanges?.[0]?.min != null ? '$${min}+' : null`
       - Image: find `ratio === '16_9'` image, fallback to first
       - ticket_link AND source_url both set to `event.url` — critical for PLAT-03 attribution
       - confidence: 1.0 (structured API data)
       - event_category: `mapTmClassification(event.classifications ?? [])`

    3. **`findOrCreateVenue(name, city, province, address): Promise<number>`:**
       - Query `db.query.venues.findFirst({ where: and(ilike(venues.name, name), eq(venues.city, city)) })`
       - If found, return `existing.id`
       - If not found, insert new venue with `{ name, address, city, province }` (lat/lng null — geocoder handles on first scrape run)
       - Return `inserted.id`
       - Import `ilike, eq, and` from drizzle-orm

    4. **`mapTmClassification(classifications): EventCategory`:**
       - Extract `segment = classifications[0]?.segment?.name?.toLowerCase() ?? ''`
       - Extract `genre = classifications[0]?.genre?.name?.toLowerCase() ?? ''`
       - Map per the table in RESEARCH.md Pattern 4

    Then create `src/lib/scraper/ticketmaster.test.ts`:
    - Mock `global.fetch` for API responses
    - Mock `@/lib/db/client` (db.query.venues.findFirst, db.insert().values().returning())
    - Mock `./normalizer` (upsertEvent)
    - Test all behaviors listed above
    - Use the same jest mocking patterns as `eventbrite.test.ts` and `bandsintown.test.ts`

    **Important:** Export `mapTmClassification` and `findOrCreateVenue` for direct unit testing (not just scrapeTicketmaster).

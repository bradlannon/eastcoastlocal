    Create `src/lib/db/merge-venue.ts` by extracting the merge algorithm from `scripts/venue-dedup-backfill.ts` lines 191-238. The function signature:

    ```typescript
    export async function performVenueMerge(opts: {
      canonicalId: number;
      duplicateId: number;
      candidateId: number;
      nameScore: number;
      distanceMeters: number | null;
      duplicateName: string;
      duplicateCity: string;
    }): Promise<{ eventsReassigned: number; eventsDropped: number }>
    ```

    CRITICAL DIFFERENCE from backfill script: Before deleting an event (unique constraint violation), first delete its `event_sources` rows. The `event_sources.event_id` FK has no ON DELETE CASCADE (Postgres default is RESTRICT), so deleting the event without cleaning up event_sources will throw a FK violation error.

    Steps inside performVenueMerge:
    1. Query all events for the duplicate venue
    2. For each event, try to reassign venue_id to canonical. On unique constraint error:
       a. Delete event_sources rows for that event_id
       b. Delete the event
    3. Bulk reassign scrape_sources from duplicate to canonical
    4. Delete duplicate venue row
    5. Insert into venue_merge_log (canonical_venue_id, merged_venue_name, merged_venue_city, name_score, distance_meters)
    6. Update venue_merge_candidates: set status='merged', reviewed_at=new Date()

    For tests, mock the `db` module. Test the logic flow — each DB call in sequence, verify correct arguments, verify event_sources cleanup before event deletion on conflict path.

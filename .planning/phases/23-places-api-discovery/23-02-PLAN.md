    - Place with no match (keep_separate) and websiteUri is staged as pending
    - Place with no match and no websiteUri is staged as no_website with synthetic URL places:{id}
    - All staged records have discovery_method='google_places'
    - All staged records have google_place_id, lat, lng, address, place_types set
    - discovery_score set from scorePlacesCandidate
    - Auto-approve fires promoteSource when score >= PLACES_AUTO_APPROVE and status is pending
    - Province venues loaded once before city loop (not per-result)
    - runPlacesDiscovery accepts city list array and orchestrates search + dedup + stage for each city
    - Per-city errors are caught and logged, not thrown (continues to next city)
    - Throttle between cities uses PLACES_THROTTLE_MS env var
  </behavior>
  <action>
    Add to `src/lib/scraper/places-discoverer.ts`:

    1. **`enrichVenue(venueId: number, place: PlaceResult): Promise<void>`** — updates existing venue row:
       - Backfills google_place_id if not already set
       - Updates address if place.formattedAddress is richer
       - Logs: `Enriched venue ${venueId} with Places data (${place.id})`

    2. **`processPlaceResult(place: PlaceResult, province: string, city: string, provinceVenues: VenueRow[]): Promise<'enriched' | 'staged_pending' | 'staged_no_website' | 'staged_review' | 'skipped'>`** — handles one place:
       - **Step 1 — google_place_id fast-path:** Query `discovered_sources` by google_place_id. If found, skip (already staged/processed). Then query `venues` by google_place_id AND province. If found, enrichVenue and return 'enriched'.
       - **Step 2 — fuzzy dedup:** Loop provinceVenues, call `scoreVenueCandidate()` with incoming `{ name: place.displayName.text, lat: place.location?.latitude ?? null, lng: place.location?.longitude ?? null }`. On 'merge': enrichVenue, return 'enriched'. On 'review': stage as 'pending' with `raw_context` containing near-match venue name, return 'staged_review'.
       - **Step 3 — staging:** Determine status: `place.websiteUri ? 'pending' : 'no_website'`. Build URL: `place.websiteUri ?? 'places:' + place.id`. Build domain: `place.websiteUri ? new URL(place.websiteUri).hostname : 'google-places'`. Insert into discovered_sources with all fields (url, domain, source_name, province, city, status, discovery_method='google_places', google_place_id, lat, lng, address, place_types as JSON.stringify, discovery_score from scorePlacesCandidate). Use `onConflictDoNothing()` for idempotency.
       - Return appropriate status string.

    3. **`runPlacesDiscovery(cities: Array<{ city: string; province: string }>): Promise<DiscoveryRunResult>`** — main orchestrator:
       - Define `DiscoveryRunResult = { citiesSearched: number; candidatesFound: number; enriched: number; stagedPending: number; stagedNoWebsite: number; autoApproved: number; errors: number }`
       - Load all venues for the province(s) in the city list once upfront: `SELECT id, name, lat, lng, google_place_id, province FROM venues WHERE province IN (distinct provinces from cities)`
       - Loop cities with throttle delay between each (skip delay on first)
       - Per city: try/catch around `searchCity()` + per-result `processPlaceResult()`. On error, log and increment errors counter, continue.
       - After all cities: auto-approve loop. Query all discovered_sources with `status='pending'` and `discovery_method='google_places'` and `discovery_score >= PLACES_AUTO_APPROVE`. Call `promoteSource()` for each. Log count.
       - Return DiscoveryRunResult with all counts.
       - Log summary: `Places discovery complete: ${result.citiesSearched} cities, ${result.candidatesFound} candidates, ${result.enriched} enriched, ${result.stagedPending} pending, ${result.stagedNoWebsite} stubs, ${result.autoApproved} auto-approved, ${result.errors} errors`

    Write tests with mocked db and fetch:
    - google_place_id match -> enriches venue, returns 'enriched'
    - fuzzy merge match -> enriches venue, returns 'enriched'
    - fuzzy review match -> stages as pending with raw_context, returns 'staged_review'
    - no match + websiteUri -> stages as pending with correct fields
    - no match + no websiteUri -> stages as no_website with synthetic URL `places:{id}` and domain `google-places`
    - all inserts have discovery_method='google_places' and google_place_id set
    - auto-approve fires for score >= 0.8 pending records
    - per-city error does not abort the run

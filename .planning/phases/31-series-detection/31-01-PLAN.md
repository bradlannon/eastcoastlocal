<feature>
  <name>Series Detection Algorithm</name>
  <files>src/lib/series-detector.ts, src/lib/series-detector.test.ts</files>
  <behavior>
    ## Pure function tests (RED first, then GREEN):

    ### performerFuzzyRatio
    - performerFuzzyRatio("open mic night", "open mic") returns a value below 0.20 (should cluster)
    - performerFuzzyRatio("jazz night", "open mic night") returns a value above 0.20 (should NOT cluster)
    - performerFuzzyRatio("", "") returns 0
    - performerFuzzyRatio("trivia night", "trivia nite") returns below 0.20

    ### hasRecurrenceKeyword
    - hasRecurrenceKeyword("Weekly Open Mic") returns true
    - hasRecurrenceKeyword("Every Tuesday Trivia") returns true
    - hasRecurrenceKeyword("Bingo Night") returns true
    - hasRecurrenceKeyword("karaoke") returns true
    - hasRecurrenceKeyword("jam night") returns true
    - hasRecurrenceKeyword("quiz night") returns true
    - hasRecurrenceKeyword("open stage") returns true
    - hasRecurrenceKeyword("John Smith Band") returns false
    - hasRecurrenceKeyword("") returns false

    ### isWeekdayRegular
    - 4 dates all on Tuesday → true (4 >= MIN_OCCURRENCES=3)
    - 3 dates on Monday, 1 on Wednesday → true (3 Mondays >= 3)
    - 2 dates on Friday → false (below MIN_OCCURRENCES)
    - Empty array → false
    - 5 dates each on a different weekday → false (no single weekday has 3+)

    ### clusterPerformers
    - ["open mic night", "open mic", "open mic nite"] clusters into one group
    - ["jazz night", "open mic night"] stays as two separate clusters
    - Representative for a cluster is the most-frequent variant (not first-encountered)
    - Single performer returns a single cluster with that performer as representative
  </behavior>
  <implementation>
    Build `src/lib/series-detector.ts` with:

    1. **Constants:** SERIES_LEVENSHTEIN_THRESHOLD (0.20), SERIES_MIN_OCCURRENCES (3), SERIES_WINDOW_DAYS (90), RECURRENCE_KEYWORDS list (include: "every", "weekly", "open mic", "trivia", "bingo", "karaoke", "open stage", "jam night", "quiz night")

    2. **performerFuzzyRatio(a, b):** Returns `distance(a, b) / Math.max(a.length, b.length)`. Handle empty strings (return 0).

    3. **hasRecurrenceKeyword(text):** Lowercase check against RECURRENCE_KEYWORDS array using `.includes()`.

    4. **isWeekdayRegular(dates):** Count occurrences of each weekday (0-6 via `getDay()`). Return true if any weekday count >= SERIES_MIN_OCCURRENCES.

    5. **clusterPerformers(normalizedPerformers):** Greedy clustering with Levenshtein. For each name, check against existing cluster representatives. If within threshold, add to cluster. Otherwise create new cluster. After clustering, reassign representative to the most-frequent variant within each cluster (per Pitfall 2 from research).

    6. **detectAndTagSeries():** The main async function:
       - SELECT all non-archived events (id, venue_id, normalized_performer, performer, description, event_date, series_id) where event_date is within SERIES_WINDOW_DAYS past to 90 days future
       - Group by venue_id
       - For each venue group, cluster performers via clusterPerformers()
       - For each cluster, check three signals:
         a. isWeekdayRegular(cluster dates) → series if true
         b. hasRecurrenceKeyword on any performer or description in cluster → series if true (keyword bypass, threshold=1 occurrence)
         c. Any event has recurrence_pattern hint (from Gemini) AND cluster has >= 2 occurrences → series
       - If any signal fires: upsert into recurring_series (venue_id, representative), then UPDATE events SET series_id for all cluster variants
       - Return { seriesUpserted: number, eventsTagged: number }
       - CRITICAL: WHERE clause for event tagging MUST include `isNull(events.archived_at)` — do not tag archived events

    Reference research patterns for Drizzle queries: 31-RESEARCH.md Pattern 3 (upsert) and code examples section.
  </implementation>
</feature>

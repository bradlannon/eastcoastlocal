   ```

   npm install -D drizzle-kit tsx jest @types/jest ts-jest jest-environment-node
   ```

3. Create `drizzle.config.ts` at project root — use the exact pattern from RESEARCH.md Pattern 3:
   - `out: './drizzle'`, `schema: './src/lib/db/schema.ts'`, `dialect: 'postgresql'`
   - Read DATABASE_URL from env via `dotenv/config` import

4. Create `src/lib/db/schema.ts` — define all three tables per CONTEXT.md locked schema decisions:
   - `venues`: id (serial PK), name, address, city, province (text — NB/NS/PEI/NL), lat (doublePrecision), lng (doublePrecision), website, phone, venue_type, created_at
   - `events`: id (serial PK), venue_id (FK to venues, NOT NULL), performer, normalized_performer, event_date (timestamp NOT NULL), event_time (text), source_url, scrape_timestamp, raw_extracted_text, price, ticket_link, description, cover_image_url, created_at, updated_at
   - `scrape_sources`: id (serial PK), url (unique NOT NULL), venue_id (FK to venues, NOT NULL), scrape_frequency (default 'daily'), last_scraped_at, last_scrape_status (default 'pending'), source_type (NOT NULL), enabled (boolean, default true — use Drizzle `boolean()` per research recommendation, NOT text), created_at
   - Composite dedup unique index on events: `(venue_id, event_date, normalized_performer)` — this column order is critical for query performance
   - Index on `event_date` for date filtering
   - Use `integer('venue_id')` for FK columns (NOT `serial` — serial auto-generates values, integer references existing ones)

5. Create `src/lib/db/client.ts` — Drizzle client singleton per RESEARCH.md Pattern 1:
   - Import `neon` from `@neondatabase/serverless`, `drizzle` from `drizzle-orm/neon-http`
   - Import all schema, pass to drizzle for relational query support
   - Export `db`

6. Create `src/types/index.ts` — export inferred types using Drizzle's `InferSelectModel` and `InferInsertModel`:
   - `Event`, `NewEvent`, `Venue`, `NewVenue`, `ScrapeSource`, `NewScrapeSource`

7. Create `src/app/api/health/route.ts` — per RESEARCH.md health check pattern:
   - GET handler that runs `db.execute(sql\`SELECT 1\`)`
   - Returns `{ status: 'ok', db: 'connected' }` on success
   - Returns 500 with `{ status: 'error', db: 'disconnected' }` on failure

8. Update `package.json` scripts — add:
   - `"db:generate": "drizzle-kit generate"`
   - `"db:migrate": "drizzle-kit migrate"`
   - `"db:studio": "drizzle-kit studio"`
   - `"db:seed": "tsx src/lib/db/seed.ts"`
   - Update `"build"` to: `"npm run db:migrate && next build"`

9. Create `.env.example` with: `DATABASE_URL=postgresql://...` (placeholder)

10. Update `.gitignore` to include `.env` and `.env.local` (create-next-app likely handles this, verify)

11. Replace default `src/app/page.tsx` with a simple placeholder: "East Coast Local — Coming Soon" heading. Minimal, no complex layout needed.

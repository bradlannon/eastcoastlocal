<objective>
Add event_sources join table for multi-source event provenance tracking, and implement non-destructive source_url conflict resolution.

Purpose: The system currently has no way to track that an event was discovered from multiple sources. This plan adds an event_sources join table (ATTR-01) and modifies upsertEvent to fill source_url only when empty (ATTR-02), enabling cross-source attribution without data loss.

Output: New event_sources table in schema + migration, extended upsertEvent with event_sources tracking and COALESCE source_url, updated callers (ticketmaster, orchestrator, bandsintown, eventbrite), passing tests.
</objective>

**schema.ts changes:**

1. Add source_type enum and event_sources table AFTER the scrape_sources definition:

```typescript
export const SOURCE_TYPES = ['scrape', 'ticketmaster', 'manual'] as const;
export const sourceTypeEnum = pgEnum('source_type', SOURCE_TYPES);

export const event_sources = pgTable(
  'event_sources',
  {
    id: serial('id').primaryKey(),
    event_id: integer('event_id').references(() => events.id).notNull(),
    scrape_source_id: integer('scrape_source_id').references(() => scrape_sources.id),
    source_type: sourceTypeEnum('source_type').notNull(),
    first_seen_at: timestamp('first_seen_at').defaultNow().notNull(),
    last_seen_at: timestamp('last_seen_at').defaultNow().notNull(),
  },
  (table) => [
    uniqueIndex('event_sources_dedup').on(table.event_id, table.source_type),
    index('event_sources_event_id_idx').on(table.event_id),
  ]
);
```

IMPORTANT: Use `uniqueIndex` on `(event_id, source_type)` NOT `(event_id, scrape_source_id)` — PostgreSQL NULL != NULL in unique indexes, and TM events always have scrape_source_id = NULL which would allow duplicates. The semantic is "one attribution record per event per source type" which is correct for v1.5 scope.

2. Run `npm run db:generate` to create the Drizzle migration SQL file, then `npm run db:migrate` to apply it.

**normalizer.ts changes:**

1. Add imports: `import { sql } from 'drizzle-orm'` and `import { event_sources } from '@/lib/db/schema'`.

2. Extend upsertEvent signature with two new optional parameters (backward-compatible defaults):

  venueId: number,
  extracted: ExtractedEvent,
  sourceUrl: string,
  scrapeSourceId: number | null = null,
  sourceType: 'scrape' | 'ticketmaster' | 'manual' = 'scrape'
): Promise<void>

3. Add `.returning({ id: events.id })` to the existing `db.insert(events)` chain to get the event ID for the join table insert.

This preserves existing source_url if non-null, fills it if null. Per user decision: non-destructive, never overwrites.

5. After the events upsert, insert into event_sources:
```typescript
const [row] = await db
  .insert(events)
  .values({ ... })
  .onConflictDoUpdate({ ... })
  .returning({ id: events.id });

await db
  .insert(event_sources)
  .values({
    event_id: row.id,
    scrape_source_id: scrapeSourceId,
    source_type: sourceType,
    first_seen_at: new Date(),
    last_seen_at: new Date(),
  })
  .onConflictDoUpdate({
    target: [event_sources.event_id, event_sources.source_type],
    set: { last_seen_at: new Date() },
  });
```

  <name>Task 1: Auth library, middleware, and login API</name>
  <files>src/lib/auth.ts, src/middleware.ts, src/app/admin/login/actions.ts, src/app/api/auth/logout/route.ts, .env.example</files>
  <action>
Install jose: `npm install jose`

1. Create `src/lib/auth.ts` with:
   - `ADMIN_EMAIL` and `ADMIN_PASSWORD_HASH` read from `process.env.ADMIN_EMAIL` and `process.env.ADMIN_PASSWORD_HASH`
   - `JWT_SECRET` read from `process.env.JWT_SECRET` (encode as Uint8Array for jose)
   - `SESSION_COOKIE_NAME = "admin_session"`
   - `SESSION_DURATION = 8 * 60 * 60` (8 hours)
   - `async function hashPassword(password: string): Promise<string>` — uses Web Crypto API SHA-256 to produce hex hash (Edge-compatible, no bcrypt needed for single admin)
   - `async function verifyPassword(password: string, hash: string): Promise<boolean>` — hashes input and compares to stored hash
   - `async function signToken(): Promise<string>` — creates JWT with jose `new SignJWT({ role: "admin" }).setProtectedHeader({ alg: "HS256" }).setIssuedAt().setExpirationTime("8h").sign(secret)`
   - `async function verifyToken(token: string): Promise<boolean>` — wraps `jwtVerify` from jose in try/catch, returns boolean
   - Export all functions plus constants

2. Create `src/middleware.ts` (Next.js root-level middleware):
   - Import `verifyToken`, `SESSION_COOKIE_NAME` from `@/lib/auth`
   - For requests matching `/admin(.*)` EXCEPT `/admin/login`:
     - Read `SESSION_COOKIE_NAME` cookie
     - If missing or `verifyToken` returns false: redirect to `/admin/login`
   - For `/admin/login`: if valid session exists, redirect to `/admin`
   - Export `config.matcher` as `["/admin/:path*"]`

3. Create `src/app/admin/login/actions.ts` (server action):
   - `"use server"` directive
   - `async function login(formData: FormData): Promise<{ error?: string }>`
   - Extract email and password from formData
   - Compare email to `ADMIN_EMAIL` (case-insensitive)
   - Call `verifyPassword(password, ADMIN_PASSWORD_HASH)`
   - If valid: call `signToken()`, set httpOnly cookie via `cookies().set(SESSION_COOKIE_NAME, token, { httpOnly: true, secure: process.env.NODE_ENV === "production", sameSite: "lax", path: "/", maxAge: SESSION_DURATION })`
   - Use `redirect("/admin")` from next/navigation on success
   - If invalid: return `{ error: "Invalid email or password" }`

4. Create `src/app/api/auth/logout/route.ts`:
   - POST handler only
   - Delete the session cookie via `cookies().delete(SESSION_COOKIE_NAME)`
   - Return `NextResponse.redirect(new URL("/admin/login", request.url))`

5. Update `.env.example` — append:
   ```
   ADMIN_EMAIL=admin@example.com
   ADMIN_PASSWORD_HASH=your-sha256-hex-hash
   JWT_SECRET=your-random-secret-at-least-32-chars
   ```

Note: Using SHA-256 via Web Crypto (not bcrypt) per single-admin-credential decision — bcrypt requires Node.js runtime and adds a native dependency. SHA-256 is sufficient for a single known credential hashed at setup time.
